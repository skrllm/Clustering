Для запуска: Необходимо открыть проект через Visual Studio, а также иметь .net Framework версии не менее 4.7.2,
в терминале PM прописать Install-Package MaterialDesignThemes.

Описание:
Приложение построено по паттерну MVVM с использованием технологии Wpf + MaterialDesign,
Существует главное окно MainWindow.xaml, В котором расположен весь Фронтенд.
Все элементы привязаны к ClusterViewModel.
Каждая кнопка выполняет свою команду.


Важно: 
1) Для работы необходимо иметь .txt файл с координатами точек (пример конструкции файла лежит в корневой папке DataExamle\PixelsCoordinates.txt 
2) Координаты пикселей должны соотвествовать картинке (координаты не должны превышать ширину и высоту изображения, иначе приложение сломается)
3) Файл должен быть разбит построчно.
4) ПО Работает в однопоточном режиме, поэтому после нажатия на кнопку "Кластеризовать" ПО зависает
на срок, зависящий от объема изначального файла.

Процесс работы: 
1) На слайдере выбираем степень "близости" в пикселях (для каждого размера изображения своя оптимальная степень)
(по умолчанию для размера 300*300 стоит степень "50" пикселей)
2) Нажав на кнопку "Выбрать изображение", выбираем необходимое изображение.
3) Нажимаем на кнопку "Выбрать список точек", выбираем текстовый файл с списком точек
4) Нажимаем на кнопку "Кластеризовать", после чего справа, на изображенни, появятся кластеры.


Описание работы приложения:

1) Приложение получает все необходимые данные ( степень "близости", изображение и список точек)
2) Приложение создает экземпляры модели и обращается к ним для кластеризации и расчетов.
3) Приложение выводит на экран полученный файл, после чего можно поменять степень "близости" и снова кластеризовать.


Точки Роста:
1) Иконки с числами изображать отдельно от изображения, а не рисуя на нем.
2) Благодрая пункту "1" сделать иконки с адаптивным размером (по умолчанию иконки размером 32*32).
3) Добавить код для отображения иконок в картинке, а не за ее пределами (такое возможно когда точка близко к краю изображения)
2) Переписать приложение на работу с несколькими потоками.

Алгоритм Кластеризации:
1) Файл считывается построчно и с помощью регулярных выражений находятся координаты каждой точки

2) Для каждой точки находятся соседи с расстоянием не более указанной степени "близости"
(Если расстояние между точками = 11.4 , а степень близости = 12, то точка будет в списках соседей)

3) Список точек сортируется по возрастанию соседей

4) (Пока каждая точка не будет в кластере)
{
	- Берется точка с самым большим кол-вом соседей и если она не в кластере, то она образует новый кластер
	- Перебираются все соседи точки и если они не в кластере, то добавляются в текущий кластер
}

5) У каждого кластера есть центр в определенной точке этого кластера. Находим центр кластера.
Находим точку в кластере с минимальной суммой расстояний до всех точех этого кластера.
Данная точка будет центром кластера.
